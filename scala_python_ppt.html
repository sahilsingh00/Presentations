ü§ù What's Similar?

  • High-level languages - No pointers or manual memory management
  • Clean syntax - Both are easy to read and write
  • Functional programming - Both support functions as values
  • Object-oriented - Classes, objects, inheritance
  • Comprehensions - List comprehensions in Python = for comprehensions in Scala
  • Big data ready - Both work great with Apache Spark
  • Rich libraries - Tons of useful packages available
Think of Scala as Python's statically-typed cousin!
3/12

üîÑ What's Different?

üêç Python

  • Dynamic typing
  • Interpreted
  • Runs directly
  • Mutable by default

‚ö° Scala

  • Static typing (but feels dynamic!)
  • Compiled to JVM bytecode
  • Runs on Java Virtual Machine
  • Immutable by default
Key insight: Everything in Scala returns a value (even if/else, try/catch)

üèóÔ∏è Classes

Simple Class

class Person: def __init__(self, name): self.name = name def greet(self): print(f"Hi, I'm {self.name}") p = Person("Alice") p.greet()
class Person(var name: String): def greet() = println(s"Hi, I'm $name") val p = Person("Alice") p.greet()
Scala constructors are built into the class definition!

üì¶ Case Classes (Data Classes)

from dataclasses import dataclass @dataclass(frozen=True) class Person: name: str age: int p = Person("Bob", 25) print(p.name) # Bob
case class Person(name: String, age: Int) val p = Person("Bob", 25) println(p.name) // Bob // Get a copy with changes val older = p.copy(age = 26)
Case classes give you: constructors, getters, toString, equals, and copy methods for free!

üéõÔ∏è Control Structures

If/Else

if age < 18: status = "minor" else: status = "adult"
val status = if age < 18 then "minor" else "adult"

For Loops

for i in range(1, 4): print(i)
for i <- 1 to 3 do println(i)
Scala's if/else returns a value - no need for separate assignment!

üéØ Pattern Matching (Super Switch)

# Python 3.10+ pattern matching match day: case 1: day_name = "Monday" case 2: day_name = "Tuesday" case _: day_name = "Other"
val dayName = day match case 1 => "Monday" case 2 => "Tuesday" case _ => "Other"
# Multiple values match number: case 1 | 3 | 5: result = "odd" case 2 | 4 | 6: result = "even"
val result = number match case 1 | 3 | 5 => "odd" case 2 | 4 | 6 => "even"
Pattern matching is much more powerful in Scala - you can match on types, extract values, and more!

üìã List Operations

Map (Transform)

numbers = [1, 2, 3] doubled = [x * 2 for x in numbers] # or doubled = list(map(lambda x: x * 2, numbers))
val numbers = List(1, 2, 3) val doubled = numbers.map(_ * 2) // or with for comprehension val doubled = for n <- numbers yield n * 2

Filter

evens = [x for x in numbers if x % 2 == 0] # or evens = list(filter(lambda x: x % 2 == 0, numbers))
val evens = numbers.filter(_ % 2 == 0) // or with for comprehension val evens = for n <- numbers if n % 2 == 0 yield n
The underscore (_) in Scala is like a placeholder for "the current item"
