Understanding the Foundation of Computer Systems

What is an Operating System?

Definition

An Operating System (OS) is a software that manages computer hardware and provides services for computer programs. It acts as a bridge between users and computer hardware.

Operating System's Role

Primary Roles:

  • Resource Manager: Allocates CPU, memory, and storage. (like hard drives) to different running programs or processes.
  • Interface Provider: Creates user-friendly interaction
  • Security Guard: Protects system and user data
  • Traffic Controller: Manages program execution

üîç Real-World Example:

When you open multiple apps on your phone:

  • OS decides which app gets CPU time
  • Allocates memory to each app
  • Prevents apps from interfering with each other
  • Manages screen display and touch input

Operating System Operations

Key Operations:

  • Process Management: Creating, scheduling, and terminating processes
  • Memory Management: Allocating and deallocating memory
  • File System Management: Organizing and accessing files
  • I/O Management: Controlling input/output devices
  • Security & Protection: User authentication and access control

üîç Example - Opening a Word Document:

1. OS loads Word program into memory
2. Creates a process for Word application
3. Allocates memory for the document
4. Manages keyboard and mouse input
5. Controls display output to screen

Functions of Operating System

Process Management

Controls program execution and multitasking

Memory Management

Allocates RAM efficiently among programs

File Management

Organizes files and folders on storage

Device Management

Controls hardware devices like printers

Security Management

Protects system from unauthorized access

Network Management

Handles internet and network connections

Computing Environments

Types of Computing Environments:

  • Desktop Computing: Personal computers for individual use
  • Server Computing: Powerful computers serving multiple users
  • Mobile Computing: Smartphones and tablets
  • Cloud Computing: Services delivered over the internet
  • Embedded Systems: Computers built into other devices

üîç Examples:

Desktop: Windows PC at home
Server: Google's web servers
Mobile: Android phone
Cloud: Netflix streaming service

Operating System Services

Services for Users:

  • Program Execution: Running applications
  • I/O Operations: Reading/writing files
  • File System Manipulation: Creating, deleting files
  • Communications: Inter-process communication
  • Error Detection: Finding and handling errors

Services for System:

  • Resource Allocation: Distributing system resources
  • Accounting: Tracking resource usage
  • Protection & Security: Controlling access

  • Graphical User Interface (GUI): Icons, windows, menus
  • Touch Interface: Touch gestures on mobile devices
  • Voice Interface: Voice commands

System Calls

What are System Calls?

System calls are programming interfaces that allow programs to request services from the operating system kernel.

How System Calls Work:

  • Program makes a request to the OS
  • OS kernel processes the request
  • Kernel performs the operation
  • Result is returned to the program

üîç Simple Example:

When a program wants to save a file:

1. Program calls "write()" system call
2. OS receives the request
3. OS writes data to the disk
4. OS returns success/failure status

Types of System Calls

1. Process Control:

  • create(), exit(), wait(), exec()
  • Example: Starting a new program

2. File Management:

  • open(), read(), write(), close()
  • Example: Opening and reading a text file

3. Device Management:

  • request(), release(), read(), write()
  • Example: Printing a document

4. Information Maintenance:

  • getpid(), alarm(), sleep()
  • Example: Getting current time

5. Communications:

  • pipe(), socket(), send(), receive()
  • Example: Sending email

Operating System Structure

Common OS Structures:

  • Monolithic Structure: All OS components in single layer
  • Layered Structure: OS divided into layers
  • Microkernel: Minimal kernel with services as processes
  • Modular Structure: Kernel with loadable modules
  • Hybrid Structure: Combination of different approaches

üîç Examples:

Monolithic: Traditional Linux
Layered: THE operating system
Microkernel: Minix
Modular: Modern Linux
Hybrid: Windows NT, macOS

Summary

Key Takeaways:

  • OS is the bridge between users and hardware
  • Main functions: Process, Memory, File, Device management
  • Provides various interfaces: CLI, GUI, Touch, Voice
  • System calls enable program-OS communication
  • Different structures suit different needs
